# Package Build & Release Automation Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `matrix-rain-webgpu` installable from npm with a Vite library build (shaders pre-transformed so consumers need no plugins), and releasable repeatably via release-it + GitHub Actions OIDC trusted publishing, entering the `1.0.0-beta` line.

**Architecture:** Vite library mode emits ESM JS with `unplugin-typegpu` pre-transforming TGSL at build time; `tsc` emits `.d.ts` separately (no `vite-plugin-dts`, to avoid an extra dep against the repo's `tsover` TS fork). `react`/`react-dom`/`typegpu`/`@typegpu/react` become peerDependencies (the fix for the duplicate-instance bug class from `91bc083`). release-it + `@release-it/conventional-changelog` drive version + `CHANGELOG.md` from conventional commits; a `workflow_dispatch` GitHub Action runs the release tokenless via OIDC (provenance automatic). The first publish is a one-time local bootstrap (OIDC can't create a package), after which trusted publishing handles everything.

**Tech Stack:** Vite 8 (library mode), `unplugin-typegpu`, `tsover` (TypeScript), release-it 20 + `@release-it/conventional-changelog` 11, GitHub Actions, npm ≥ 11.5.1 (OIDC trusted publishing).

**Verification approach (READ FIRST):** This project has a standing policy of **no unit-test framework during v1** (see memory `project-testing-deferred`: tsc + oxlint + build inspection + manual eyeballing only). Do **not** add vitest/Playwright or write `*.test.ts`. Each task is verified by: `tsc` typecheck, `oxlint`/`oxfmt`, a successful `vite build`, **inspection of `dist/` output**, `release-it --dry-run` output, and the docs site still building. Steps below give the exact command + expected output to check.

**Working mode:** This is Mode B work (build/release — see memory `feedback-collaborative-coding`). Execute on a branch with a **draft PR** for the maintainer to review and merge; do not push to `main` directly. Tasks marked **[MAINTAINER]** are manual steps the maintainer runs on their own npm/GitHub account — the agent prepares and documents them but does not perform them.

**Spec:** `planning/superpowers/specs/2026-06-13-package-build-and-release-design.md`

### Task 1 [SPIKE]: Prove the Vite library build pre-transforms shaders

**Files:**

- [ ] **Step 1: Install the type-emit + build deps that aren't present**

Run:

- [ ] **Step 2: Create `vite.config.ts`** (mirror the docs plugin usage exactly — `unplugin-typegpu/vite`, duck-typed cast)

```ts
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';
import typegpu from 'unplugin-typegpu/vite';

// Library build for npm. unplugin-typegpu transforms the `'use gpu'` TGSL at
// BUILD time (Babel), so the published JS is self-contained — consumers need
// no TypeGPU build plugin. Peers are externalized so the consumer provides a
// single instance (see the 91bc083 duplicate-instance bug).
export default defineConfig({
  // eslint-disable-next-line @typescript-eslint/no-explicit-any
  plugins: [react(), typegpu({}) as any],
  build: {
    target: 'esnext',
    lib: {
      entry: 'src/index.ts',
      formats: ['es'],
      fileName: () => 'index.js',
    },
    rollupOptions: {
      external: [
        'react',
        'react-dom',
        'react/jsx-runtime',
        /^@typegpu\//,
        'typegpu',
      ],
    },
  },
});
```

NOTE: `/^@typegpu\//` externalizes `@typegpu/react` AND `@typegpu/noise`. The spike (Step 5) decides whether `@typegpu/noise` should instead be bundled+inlined; if the transform inlines it there will simply be no `@typegpu/noise` import left, making the external entry a harmless no-op.

- [ ] **Step 3: Create `tsconfig.build.json`** (declaration-only emit)

```json
{
  "extends": "./tsconfig.app.json",
  "compilerOptions": {
    "noEmit": false,
    "declaration": true,
    "emitDeclarationOnly": true,
    "outDir": "dist",
    "rootDir": "src"
  },
  "include": ["src"],
  "exclude": ["src/demo"]
}
```
(`tsconfig.app.json` likely sets `noEmit: true` for typechecking; this overrides it. Confirm `src/demo` is excluded if present — the demo must not ship.)

- [ ] **Step 4: Run the JS build**

Run:
```bash
npx vite build
```
Expected: `dist/index.js` produced, no errors. If `unplugin-typegpu` errors, STOP and report — this is the spike's whole point.

- [ ] **Step 5: Inspect `dist/index.js` for surviving TGSL and the noise import**

Run:
```bash
grep -c "use gpu" dist/index.js || echo "0 (good: no raw directives)"
grep -n "@typegpu/noise" dist/index.js || echo "noise NOT imported at runtime → devDependency"
```
Expected & decision:
- `use gpu` count should be **0** (the directives were transformed away). If non-zero, the transform isn't running at build time → STOP and report.
- If `@typegpu/noise` is **not** imported → it's build-time-only → classify as **devDependency** (Task 3).
- If it **is** imported → classify as **peerDependency** (Task 3) and keep it externalized.

- [ ] **Step 6: Emit + inspect the declarations**

Run:
```bash
npx tsc -p tsconfig.build.json
ls dist/index.d.ts && head -20 dist/index.d.ts

Expected: `dist/index.d.ts` exists and re-exports `MatrixRainWebGPU`, `isWebGPUSupported`, and the option types. If `tsover` fails to emit declarations, fall back: add `typescript` (real) as a devDep used only for this emit, and report the deviation.

- [ ] **Step 7: Plugin-less consumer smoke check**

Create a throwaway sibling dir, `npm pack` the current build, install the tarball into a bare Vite+React app with **no** `unplugin-typegpu`, import `MatrixRainWebGPU`, and `vite build` the consumer.

### Task 6: GitHub Actions release workflow (OIDC trusted publishing)

**Files:**
- Create: `.github/workflows/release.yml`

- [ ] **Step 1: Create the workflow**

```yaml
name: Release

on:
  workflow_dispatch:
    inputs:
      increment:
        description: "Which release to cut"
        type: choice
        options: [beta, major-beta, patch, minor, major]
        default: beta
      dry_run:
        description: "Dry run (no publish/tag)"
        type: boolean
        default: false

permissions:
  contents: write      # push the bump/changelog commit + tag, create the GH release
  id-token: write      # OIDC token for trusted publishing (+ automatic provenance)

jobs:
  release:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0           # full history for conventional-changelog
      - uses: actions/setup-node@v4
        with:
          node-version: 22
      - run: npm install -g npm@latest    # trusted publishing needs npm >= 11.5.1
      - run: npm ci
      - name: Configure git identity
        run: |
          git config user.name  "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
      - name: Map increment to release-it args
        id: args
        run: |
          case "${{ inputs.increment }}" in
            beta)        echo "args=--preRelease=beta" >> "$GITHUB_OUTPUT" ;;
            major-beta)  echo "args=major --preRelease=beta" >> "$GITHUB_OUTPUT" ;;
            *)           echo "args=${{ inputs.increment }}" >> "$GITHUB_OUTPUT" ;;
          esac
      - name: Release
        run: npx release-it ${{ steps.args.outputs.args }} ${{ inputs.dry_run && '--dry-run' || '' }} --ci
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          # No NPM_TOKEN — OIDC trusted publishing authenticates the publish.
```

- [ ] **Step 2: Lint the YAML mentally + commit** (the real validation is the dry-run dispatch in Task 7)

```bash
git add .github/workflows/release.yml
git commit -m "ci: release workflow via OIDC trusted publishing (workflow_dispatch)"
```

### Task 8: RELEASE.md maintainer runbook

**Files:**
- Create: `RELEASE.md`

- [ ] **Step 1: Write the runbook** covering every item in spec §6. Use this skeleton, filling in the spike-confirmed specifics:

```markdown
# Releasing matrix-rain-webgpu

This is the maintainer runbook. Releases run from GitHub Actions via OIDC
trusted publishing (no npm token stored). Versions and `CHANGELOG.md` are
derived from conventional commits (`feat:` → minor, `fix:` → patch,
`feat!:`/`BREAKING CHANGE:` → major), so write good commit messages.

## One-time setup (do this once)

1. **Bootstrap the first publish locally.** OIDC cannot create a package that
   doesn't exist yet, so the very first version is published from your machine:
   ```bash
   npm login                       # authenticated session (2FA)
   npx release-it major --preRelease=beta --dry-run   # verify first!
   npx release-it major --preRelease=beta             # publishes 1.0.0-beta.0
   ```
2. **Enable trusted publishing.** On npmjs.com → the `matrix-rain-webgpu`
   package → Settings → Trusted Publishing: add a GitHub Actions publisher for
   repo `chicio/matrix-rain-webgpu`, workflow `release.yml`. Select the
   "publish" allowed action. After this, all releases run from CI tokenless.

## Cutting a release (every time)

1. **Always dry-run first.** Actions → Release → Run workflow → set
   `dry_run: true`, pick the increment. Confirm the computed version, the
   `CHANGELOG.md` diff, and the publish plan look right.
2. **Run for real** with the same increment and `dry_run: false`:
   - First beta on the 1.0.0 line: `major-beta` → `1.0.0-beta.0` (npm `beta` tag).
   - Subsequent betas: `beta` → `1.0.0-beta.1`, `.2`, … (npm `beta` tag).
   - Graduate to stable: `minor`/`patch`/`major` as appropriate → lands on `@latest`.

## What happens automatically

Version bump, `CHANGELOG.md` update, the bump commit + git tag pushed to
`main`, the GitHub release created with the changelog body, npm publish, and a
signed provenance attestation. Do none of this by hand.

## Gotchas

- The bump commit also triggers the docs `deploy` workflow — expected, harmless.
- Trusted publishing needs npm ≥ 11.5.1; the workflow upgrades npm itself.
- If `main` ever gets branch protection with required reviews, the bot push from
  release-it needs an exemption.
```
